This lesson closes out the file-and-network-surface category: what's shared
out over the network, what shouldn't be sitting on disk, and one small text
file that's more dangerous than it looks.

## SMB shares — keep the defaults, remove the rest

```
Get-SmbShare
Remove-SmbShare -Name "BadShare" -Force
```

Every Windows install creates a handful of shares automatically for
administrative and OS functions:

| Share | Purpose |
|---|---|
| `C$` | Administrative access to the C: drive root |
| `ADMIN$` | Administrative access to the Windows installation directory |
| `IPC$` | Inter-process communication, used by remote administration tools |
| `print$` | Present only if printer sharing is enabled |

**These are supposed to be there — don't remove them.** Removing a default
admin share doesn't make the machine more secure in any way scoring cares
about, and can break legitimate remote administration. The actual finding
here is any *non-default* share — something like `TeamDocs` or a bare drive
share someone set up — that isn't called out as required on the README.
Audit the full `Get-SmbShare` output, keep the four defaults above, and
remove anything else that isn't authorized.

## Prohibited files

```
Get-ChildItem C:\Users -Include *.mp3,*.mp4,*.avi,*.wav -Recurse -ErrorAction SilentlyContinue
```

Media files and similarly out-of-scope content sitting in user directories
are a standard scored finding on many images. This is a straightforward
find-and-remove item, but the search itself is worth running deliberately
rather than skimming folders by eye — `-Recurse` across every user profile
catches things a manual look would miss.

## The hosts file

```
type C:\Windows\System32\drivers\etc\hosts
```

The hosts file at `C:\Windows\System32\drivers\etc\hosts` maps hostnames
directly to IP addresses, and — critically — it's checked *before* DNS. A
clean, untampered file contains only comment lines (starting with `#`) and
possibly one active `127.0.0.1 localhost` entry. Anything else — a real,
uncommented line mapping some domain to an IP — is worth scrutinizing
closely.

The danger isn't theoretical: an entry like

```
93.184.216.34    windowsupdate.microsoft.com
```

would silently redirect all traffic meant for that domain to the listed IP,
with no error, no warning, and nothing that looks like a running "service"
to notice. It's one line in a text file, but it has the same practical
effect as a working DNS-hijack. Because it's so cheap to check (`type` the
file, read it, done), it belongs in the same quick-pass category as checking
the Run keys — a thirty-second look that closes off a real attack path.
