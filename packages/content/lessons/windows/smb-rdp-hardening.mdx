SMB and RDP are both services that let something reach into the machine from
the network — which is exactly why they're a recurring category on any
hardening checklist, and why "disable it" and "harden it" are both valid
answers depending on what the README actually needs.

## SMBv1: disable it — this is the WannaCry protocol

SMB (Server Message Block) is the protocol behind Windows file and printer
sharing. Version 1 is decades old and carries known, severe vulnerabilities —
most famously EternalBlue (MS17-010), the exploit WannaCry used to spread
itself across networks with no user interaction required. Modern Windows
doesn't need SMBv1 for normal file sharing; SMBv2 and SMBv3 handle it.

```
Get-WindowsOptionalFeature -Online -FeatureName SMB1Protocol
Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol -NoRestart
```

The first line checks whether it's currently enabled; the second turns it
off. `-NoRestart` avoids forcing an immediate reboot mid-round — a restart
will still be needed for the change to fully take effect, but you control
when.

## RDP: disable, or require NLA — read the README first

Remote Desktop Protocol lets someone open a full graphical session over the
network on port 3389. There are exactly two correct outcomes here, and which
one applies depends entirely on the README:

**If RDP is not required**, disable it outright:

```
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 1 /f
```

`fDenyTSConnections=1` turns off Remote Desktop connections entirely.

**If RDP is required**, don't disable it — that breaks a required service
and costs a penalty. Instead, force Network Level Authentication:

```
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v UserAuthentication /t REG_DWORD /d 1 /f
```

`UserAuthentication=1` means a connecting client must authenticate *before*
a full remote session is established, rather than getting to a login screen
first and authenticating after. It closes off a meaningful chunk of RDP's
attack surface (including some pre-authentication exploits) while keeping
the service the README needs.

## Why this category matters beyond the checklist

Every item elsewhere in this pillar — accounts, policy, services — describes
something a local attacker (or a local misconfiguration) could exploit.
SMB and RDP are different: they're doors that face outward, reachable
directly from the network without any other foothold on the machine first.
That's not just competition-scoring logic — RDP and SMB show up constantly
in real-world breach and ransomware reports for exactly this reason.
Treating "what's listening and reachable" as a first-pass priority, not just
another line item, is a habit worth building here.
